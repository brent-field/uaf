# 008 — Layout Inspector: Interactive Typographic Debugging UI

## Goal

Add an interactive inspector to the Layout View so that clicking or
hovering over any text block shows its typographic properties (font
family, size, weight, style, colour, position, dimensions, rotation).
This turns the Layout View into a debugging tool for verifying extraction
accuracy against the original PDF.

## Scope

Implement the inspector as a **pure client-side feature** — JavaScript
reads data already in the DOM (inline styles + new `data-*` attributes).
No new API routes, no modifications to `pdf_format.py`. The only backend
change is adding `data-*` attributes to the HTML generated by
`doc_lens.py`.

## Current Architecture (read these files first)

### How the layout view is served

1. User clicks "Layout" button in editor → HTMX sends
   `GET /artifacts/{id}/blocks?mode=layout` targeting `#doc-content`
2. Route in `src/uaf/app/frontend/routes.py` calls
   `DocLens().render_layout()` → returns `LensView.content` (raw HTML)
3. Content is injected into `partials/doc_layout.html`:
   ```html
   {% if layout_html %}
   <div class="layout-view">
       {{ layout_html | safe }}
   </div>
   {% else %}
   <div class="empty-state">...</div>
   {% endif %}
   ```
4. The partial replaces `#doc-content` innerHTML in `editor.html`

### Current HTML output from `_render_layout_node()`

Each layout block currently renders as:

```html
<div data-node-id="{uuid}" class="layout-block"
     style="position: absolute; left: 72.0pt; top: 100.0pt;
            width: 468.0pt; z-index: 994;
            font-family: &quot;Times New Roman&quot;, Times, serif;
            font-size: 10.0pt; font-weight: bold; color: #000000;
            transform: rotate(-90.0deg); transform-origin: top left">
  Text content here
</div>
```

Properties available in inline style: position, width, z-index,
font-family, font-size, font-weight, font-style, color, transform.

Properties NOT currently in the DOM: page number, reading order, height,
rotation angle (as a number), node type, first_line_weight, original PDF
font name.

## Design

### Interaction model

1. **Hover**: Light outline (2px dashed blue) around the block. A compact
   tooltip near the cursor shows font-family and size.
2. **Click**: Persistent inspector panel slides in from the right showing
   full properties. Clicking another block switches. Click same block or
   press Escape to dismiss.
3. **Panel content** (grouped):

   **Position & Size**
   - Page, Position (x, y), Width, Height, Reading order

   **Typography**
   - Font family (CSS stack), Size, Weight, Style, Colour
   - First-line weight (if applicable)

   **Rotation**
   - Angle, Transform origin

   **Content**
   - Text preview (first 200 chars, `\n` shown as `↵`)
   - Line count, Character count

   **Identity**
   - Node ID (copyable), Node type

### Visual design

- Panel: 280px wide, fixed position right side, dark background
  (`rgba(0,0,0,0.85)`), light text, monospace values.
- Tooltip: small floating div, shows `"Times New Roman" | 10pt | bold`
- Selected block: solid blue border, faint blue background tint.
- Panel must NOT shift the layout — it overlays.

## Implementation Steps

### Step 1: Add data attributes to layout block divs

**File:** `src/uaf/app/lenses/doc_lens.py`, method `_render_layout_node()`

Add data attributes for properties not already in the style string.
The node type can be determined by matching the node:

```python
# Determine node type name
node_type = "unknown"
match node:
    case Heading():
        node_type = "heading"
    case Paragraph():
        node_type = "paragraph"
    case CodeBlock():
        node_type = "code_block"
    case TextBlock():
        node_type = "text_block"
    case Image():
        node_type = "image"

# Build data attributes
data_parts = [f'data-node-id="{nid}"']
data_parts.append(f'data-node-type="{node_type}"')
if layout.page is not None:
    data_parts.append(f'data-page="{layout.page}"')
if layout.reading_order is not None:
    data_parts.append(f'data-reading-order="{layout.reading_order}"')
if layout.height is not None:
    data_parts.append(f'data-height="{layout.height}"')
if layout.rotation is not None:
    data_parts.append(f'data-rotation="{layout.rotation}"')
if layout.first_line_weight:
    data_parts.append(f'data-first-line-weight="{escape(layout.first_line_weight)}"')
data_attr_str = " ".join(data_parts)
```

Update the return to use `data_attr_str` instead of inline `data-node-id`.

### Step 3: Create `layout-inspector.js`

**File:** `src/uaf/app/static/layout-inspector.js`

**Critical HTMX consideration:** The layout view is loaded dynamically
via HTMX `hx-get`. When the partial replaces `#doc-content`, any
event listeners attached to elements inside it are destroyed. There are
two approaches:

**Approach A (recommended): Event delegation on `#doc-content`**

```javascript
document.addEventListener("DOMContentLoaded", () => {
    const root = document.getElementById("doc-content");
    if (!root) return;

    // Delegated listeners survive HTMX swaps
    root.addEventListener("mouseover", (e) => {
        const block = e.target.closest(".layout-block");
        if (block) showTooltip(block, e);
    });
    root.addEventListener("mouseout", (e) => {
        const block = e.target.closest(".layout-block");
        if (block) hideTooltip();
    });
    root.addEventListener("click", (e) => {
        const block = e.target.closest(".layout-block");
        if (block) {
            e.stopPropagation();
            showInspector(block);
        }
    });
});
```

Because `#doc-content` itself is never replaced (only its innerHTML is
swapped), delegated listeners on it persist across HTMX swaps.

**Approach B: Use HTMX afterSwap event**

```javascript
document.body.addEventListener("htmx:afterSwap", (e) => {
    if (e.detail.target.id === "doc-content") {
        initInspector();
    }
});
```

Key JS functions:
- `showTooltip(blockEl, mouseEvent)` — read computed style + data attrs,
  position tooltip near cursor
- `hideTooltip()`
- `showInspector(blockEl)` — populate panel with all properties, add
  `.selected` class to block, show panel
- `hideInspector()` — remove `.selected`, hide panel
- `formatRow(label, value)` — return HTML for one inspector row

To read properties from the DOM:

```javascript
function getBlockProperties(el) {
    const style = el.style;  // inline style object
    return {
        nodeId: el.dataset.nodeId,
        nodeType: el.dataset.nodeType || "unknown",
        page: el.dataset.page,
        x: style.left,        // "72.0pt"
        y: style.top,         // "100.0pt"
        width: style.width,   // "468.0pt"
        height: el.dataset.height,
        readingOrder: el.dataset.readingOrder,
        fontFamily: style.fontFamily,
        fontSize: style.fontSize,
        fontWeight: style.fontWeight || "normal",
        fontStyle: style.fontStyle || "normal",
        color: style.color || "#000000",
        rotation: el.dataset.rotation,
        firstLineWeight: el.dataset.firstLineWeight,
        text: el.textContent,
    };
}
```
